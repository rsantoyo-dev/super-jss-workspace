<Title>[sj] Directive Basics</Title>

<Description>
[sj] starts with plain JS objects, then evolves naturally to the sj Root API. The API returns SjStyle objects you compose in an array, unlocking autocomplete, tokens, responsive values and pseudo‑states.

Key ideas:

- [sj] accepts a single SjStyle object or an array of them.
- The sj Root API functions (e.g., `sj.padding(...)`, `sj.bg(...)`) each return an SjStyle.
- Mix plain CSS values and theme tokens; both are valid.
- Pseudo‑selectors work either as plain objects (`'&:hover': { ... }`) or via helpers (`sj.hover([...])`).
</Description>
